L’inspiration pour ce site était simplement le tout premier site web (remis à disposition par le CERN) créé par Tim Berners-Lee en 1991. Le site est hiérarchisé avec des titres par le HTML, les paragraphes sont balisés comme tel. Il n’y a pas d’images, pas de multimédia, simplement du texte et des liens.

En effet, les premières normes HTML ont été créées pour éditer des documents textuels et les lier entre eux sur le réseau du web par des hyperliens. La plupart des premiers éléments HTML sont tirés du SGML (Standardized Generalized Markup Langage), le but était d’attirer cette communauté et d’y ajouter le réseau par hyperlien. Ce n’est qu’à partir de 1993 que le terme HTML commence à apparaître .

Pour le bien de cette expérience je suis obligée d’être anachronique pour des raisons techniques. Par exemple les pages HTML utilisent l’UTF-8 pour encoder du texte, or ce langage commence à être utilisé au milieu des années 90. Cependant puisque je calque les anciennes méthodes sur les technologies d’aujourd’hui, je suis obligée de l’utiliser. Pareillement, le premier site a été codé sur l’ordinateur NeXT, et le premier site ne proposait pas de police particulière, seulement la police du navigateur par défaut. Pour l’ordinateur Next, c’était Helvetica . Donc même s’il n’y avait pas d’éléments de style dans le code, j’ai utilisé du CSS.

L’Universal Character Set Transformation Format en 8 bits est une technologie d’encodage des signes de différentes langues, permettant notamment l’uniformisation et l’interopérabilité des données entre les sites web. Concrètement, chaque signe Unicode est codé sous la forme d’une séquence d’un à quatre octets. Cette technologie est créée en 1992/93, donc le site de 1991 ne l’utilise pas. Cependant aujourd’hui, et depuis le début des années 2000, la structure d’une page HTML correcte contient la balise dans le head, pour indiquer que cette technologie est utilisée. En effet, cette balise meta charset permet de spécifier le codage des caractères, puisqu’il existe d’autres normes qui ont perdu de leur popularité quand l’UTF-8 se démocratisait, notamment ASCII . Plus que la cohérence, le but du projet est avant tout d’être accessible, donc il est nécessaire de faire des pages HTML correctes.
